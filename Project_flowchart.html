AI-Driven QA process — System Flow

One QA Manager carries a sprint end to end — read follows the arrows: the orchestrator invokes a skill, the skill delegates to a sub-agent and/or calls an MCP server, and some steps produce an output. Every phase boundary is an explicit gate.

Entry & Orchestrator

The user hands a sprint/PBI to the orchestrator, who wears two hats.

gateMode GateNormal (default) or Deep — named when the PBI is submitted; sets the scope for every skill invoked downstream.
🧭
Orchestrator

Reads the request, invokes skills, and switches hats at phase boundaries.

📘
Skill

A repeatable procedure — the how-to for one job, run the same way every time.

🤖
Sub-agent

A specialist a skill delegates real work to — reasoning or engineering.

🔌
MCP

The connector to Azure DevOps, Playwright, and Appium.

🧭 QA Manager hat

Owns Phase 1 & 2 — delegates to reasoning sub-agents to analyze the PBI, design test coverage, and prepare documents.

🛠️ SDET hat

Owns Phase 3 — delegates to the engineering sub-agents to build and run the automated tests.

MCP Servers — .mcp.json

playwrightWeb UI inspection, manual screenshots, web automation.
appiumMobile UI inspection + locator extraction for the app.
