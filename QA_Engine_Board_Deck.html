How it's built

Four simple building blocks

🧭

The Director

Plans the work, gives instructions, and checks the results — pausing for human approval at key steps.

🤖

Agent

A specialist that handles one kind of job — for example, designing tests or writing automation.

📘

Skill

A ready-made playbook of instructions that keeps every task done the same, correct way.

🔌

Connector

A secure link to our tools, like Azure DevOps and the live application.

Who's in charge

The Director wears two hats

The same lead switches roles depending on the stage of work — clearly, and one at a time.

🧭 QA Manager hat

Delegates to the QA Agent and Document Agent to plan the tests and prepare the documents.

  • Signs off coverage
  • Owns quality
🛠️ SDET hat

Delegates to the Automation Agents and QC Agent to build, run, and report on the tests.

  • Runs tests on the live app
  • Logs the defects

Looking ahead

Future Enhancements

🎯

Running the Automation Phase on different environments for different goals — for example, running Regression Cycle 1 on the Staging environment instead of a full cycle run on the QA server.

🎨

Integrating Figma as a verification step for the UI, to help generate clearer automation steps during script creation.

🔗

Expanding AI-driven tasks to include API testing by automating API calls — a major addition planned for the integration phase.

Honest view

Challenges

🔒

Running the Automation Phase on QA, Staging, and UAT environments hosted on a remote server requires a VPN connection (e.g., Asiacell).

⏳

For large-scale projects, the subscription token may be exhausted before the task is completed.

🍎

Running Appium on iOS IPA builds may require a Mac device.
